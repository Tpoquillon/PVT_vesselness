Prepare data for crosse validation experiments. From a set of boxes, in wich points have been labelled build a cross validation set, with datas balanced between classes and between boxes.

Parameters

  • annotations_file (list) – Paths to the annotation file: 3d array where points from each class i have a value of i and point without class 0. The first class has to be the vessels annotations (or the object of interest to be segmented).

  • mask_file (str, optional) – path to the mask file. the default is “auto” for a mask covering all the image

  • box_file_list (list, otional) – list of path to boxe_file, boxe in wich annotations have been performed. the default is “auto” for a box covering all the image

  • vessel_prop_list (list, optional) – list of vessel (first class) proportions. For each proportion, a cross-validation set is built. However all c-v sets are built from the same seeds. the default is [0.5, 0.4, 0.3]

  • point_per_boxe (int, optional) – number of annoted point extraced from each boxes to build the datas.

Returns

  • cross_validation_sets (dic) – dictionary of list of cross-validation set (list of mask). each key is a vessel proportion

pvtvesselness.vesselness.FeaturesOptimisation(cross_validation_dataframes, hyperparameters=None, optimise=False, sorting_by='oob + std')¶

A methode that evaluate the best set of features for the random forest classifier, by remooving worst features one by one. This method is not optimal and ImprouvedFeatureOptimaqation is recomended

Parameters

  • cross_validation_dataframes (dict) – dictionnary of cross validation dataframes.

  • hyperparameters (dict) – Optimised parameters for random forest.

  • optimise (TYPE, optional) – if we are to do sub optimisation insaide the main loop. The default is False.

  • sorting_by (string, optional) – the methode to sort features in order to remove the worst one, if “oob”, features are sorted using the default rank of sklearn random forest, using out of bag score. if “shap”, features are sorted using the mean shap rank, using shapley values. If +std is added, feature are sorted with their mean + std score and not their mean score. The default is oob + std

Returns

a dictionary of metrics showing the evolution of these one with the number of features.

Return type

dict
